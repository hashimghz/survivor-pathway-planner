# models/AGENTS.md — Contracts

## Status: FROZEN after Day 0

The pydantic models in this directory are the contract every agent depends on. Changes break parallel work.

## Change procedure

1. Open a PR with the `contract-change` label.
2. Describe the schema change and which layer triggered it.
3. All three agent owners (or their humans) acknowledge in PR review.
4. The CI schema-hash check is updated in the same PR (`make update-schema-hash`).
5. Golden fixtures in `fixtures/golden_outputs/` are regenerated by the PR author.

No agent edits this directory without the procedure above. If a layer needs a field that doesn't exist, surface the gap in PR review — don't patch around it locally.

## Rules for this directory

- Pure pydantic. No business logic, no I/O, no imports from `engine/`, `app/`, `core/`, `db/`, `data/`.
- Every field has `Field(description=...)` or a clarifying docstring on the model.
- Enums are exhaustive. If a real value isn't in the enum, the pipeline must fail loud — never coerce.
- No optional fields without a documented reason. Prefer required + sentinel values over `Optional`.
- All numeric fields with bounded ranges (confidence, levels, percentages) carry their `ge`/`le` constraints in the schema itself, not in downstream validation code.
